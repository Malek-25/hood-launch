THE PRODUCT

One primitive.
One permanent rule.

HOOD//LAUNCH is a factory for factories. It creates token launchers, and the rule is inherited forever: launched tokens must establish a market against the exact `$HOODIE` contract.

01

Create a launcher

Anyone creates one. There is no approval list, owner, or configuration ceremony.

02

Set the launch

Choose a name, ticker, fixed supply, token allocation, and the `$HOODIE` liquidity to seed.

03

Launch a market

The token, pool, reserves, and LP shares are created together—or the transaction reverts.

NOT A UI PROMISE

The hood is
hard-coded.

FAQ

Can a launcher use another quote asset?

No. There is no quote-token parameter anywhere in the launch flow. Each launcher inherits the exact `$HOODIE` contract from the factory.

Who controls the protocol?

No one has an admin switch, upgrade method, or alternative quote-asset setting in these contracts. The system is permissionless.

Where does liquidity go?

The creator supplies `$HOODIE`; the launcher creates the pool and forwards the remaining token allocation and LP shares to the selected recipient in the same transaction.
